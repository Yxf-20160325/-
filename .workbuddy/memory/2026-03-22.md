# 2026-03-22 工作日志

## 修复：`11.js` 加载报 `SyntaxError: Identifier 'users' has already been declared`

**根本原因**：插件文件里有两行完全相同的 `const { users, rooms, messages } = context;`（创建时模板自动加了一行，用户代码里又有一行）。

**修复方案（3处）**：
1. `plugins/11.js`：直接删掉重复的那行及多余注释
2. `server.cjs` 创建插件模板：去掉模板里自动注入的 `const { users, rooms, messages } = context;` 和 `// 插件初始化代码` 两行，改为直接嵌入用户代码（由用户代码自己负责写这行）
3. `public/index.html` 编辑器默认代码：在默认示例代码开头补上 `const { users, rooms, messages } = context;`，与模板改动保持一致

## 修复：导入插件无效（两个 Bug）

**Bug 1 — 导出时正则提取 code 可能为空**：原正则 `/init: function\(app, io, context\) \{([\s\S]*?)\n    \},/` 对缩进格式敏感，匹配失败时 `pluginCode` 为空字符串，导出的 JSON 里 `code` 字段为空。

**修复**：改用**括号计数法**精确定位 `init` 函数体的起止位置，无视缩进格式。提取后还会自动剥离模板注入的 `const { users, rooms, messages } = context;` 和 `// 插件初始化代码` 这两行。

**Bug 2 — 导入时 `context` 解构重复声明**：导入 API 的模板里已有 `const { users, rooms, messages } = context;`，而插件原始代码里也有一份（创建插件时模板自动加的），导致 `SyntaxError: Identifier 'users' has already been declared`，插件加载报错。

**修复**：导出时剥离模板行，导入时模板仅保留一份 `context` 解构，用户代码直接跟在其后。

**修复文件**：`server.cjs`（导出 API 约第 900 行，导入 API 约第 1004 行）

## 修复：自己创建的插件不加载

**根本原因**：Node.js 的 `require()` 有模块缓存机制，同一文件路径首次加载后会缓存模块对象，后续再 `require()` 同一路径直接返回缓存，**不读取文件**。因此：
- 创建新插件后调用 `loadPlugins()` 时，虽然文件已写入磁盘，但 `require()` 命中缓存，不执行新代码
- `reload` 操作同样无效

**修复方案**：在 `loadPlugins()` 的 `require()` 调用之前，先检查并清除 `require.cache` 中对应路径的缓存：
```js
if (require.cache[require.resolve(pluginPath)]) {
    delete require.cache[require.resolve(pluginPath)];
}
```

**修复文件**：`server.cjs`（`loadPlugins` 函数，约第 668 行）

## 新增功能：插件导出/导入

### 需求
用户希望自己创建的插件只在本地加载，并增加导出、导入插件功能。

### 实现方案

**服务端 (`server.cjs`)**
- 新增 `GET /api/plugins/:name/export`：读取插件文件，提取代码和元数据，返回 JSON 文件（Content-Disposition: attachment）
- 新增 `POST /api/plugins/import`：接收 JSON 格式的插件数据，写入 plugins 目录，执行安全检查（与创建插件相同），调用 loadPlugins() 热加载

**前端 (`public/index.html`)**
- 插件列表渲染：非内置插件（`!plugin.isBuiltin`）旁边增加绿色"导出"按钮
- 插件管理面板按钮区域增加绿色"导入插件"按钮 + 隐藏 `<input type="file" accept=".json,.plugin.json">`
- 新增函数：
  - `exportPlugin(name)`：调用导出 API，blob 下载为 `xxx.plugin.json`
  - `triggerImportPlugin()`：触发文件选择
  - `handleImportPluginFile(file)`：读取 JSON 文件，调用 `/api/plugins/import`
- `initPluginManagement()` 中绑定导入按钮和文件 input 事件

### 插件文件格式（.plugin.json）
```json
{
  "name": "plugin_name",
  "description": "描述",
  "version": "1.0.0",
  "code": "用户代码段",
  "exportedAt": "ISO时间",
  "exportedBy": "chatroom-plugin-system"
}
```

### 注意
- 内置插件（weather 等）不支持导出（返回 403）
- 导入时若同名插件已存在会直接覆盖
- 代码安全检查与创建插件一致
