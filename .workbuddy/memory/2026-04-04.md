## 修复位置发送不出来位置的 Bug

**问题**：用户点击发送位置按钮后，无法获取到位置信息。

**根本原因**：服务器 `server.cjs` 第 123 行设置了 `Permissions-Policy: geolocation=()`，这会禁止所有页面使用地理位置 API（包括同源页面）。浏览器会直接拒绝 `navigator.geolocation.getCurrentPosition()` 调用，导致无法获取位置。

**修复方案**：
将 `geolocation=()` 改为 `geolocation=(self)`，允许同源页面使用位置 API。

```javascript
// 修复前
res.setHeader('Permissions-Policy', 'camera=(self), microphone=(self), geolocation=()');

// 修复后
res.setHeader('Permissions-Policy', 'camera=(self), microphone=(self), geolocation=(self)');
```

**修复文件**：`server.cjs` 第 123 行

**验证方法**：重启服务器后刷新页面，再次尝试发送位置功能应该正常工作。

## 修复通知永久不消失 Bug

**问题描述**：文档删除等通知弹出后，如果用户点击了通知，通知永远不会消失，一直停留在页面顶部。

**根本原因**：`showNotification()` 函数的点击事件处理逻辑有缺陷：
- 当用户点击通知时，只调用了 `clearTimeout(timer)` 取消自动消失定时器
- 但没有移除 DOM 元素本身！
- 对于没有 callback 的通知（如文档删除通知），点击后定时器被清除，通知就永远留在了页面上

**修复方案**：重写点击事件处理逻辑，统一为：
1. 清除定时器
2. 如果有 callback 则执行 callback
3. **始终执行淡出动画并移除 DOM 元素**
4. 移除旧的独立 callback 事件监听器（避免重复绑定和冲突）

**修复文件**：`public/index.html` 第 5677-5704 行（`showNotification` 函数）

## 修复消息撤回功能失效

**问题描述**：用户点击"撤回"按钮后，消息没有被撤回，也没有任何反馈。

**根本原因**：
1. 服务器端在撤回失败时（如无权限、消息不存在、不是自己的消息）没有发送反馈给客户端
2. 客户端没有监听 `permission-denied` 和 `recall-failed` 事件
3. 客户端 `recallMessage` 函数没有检查 socket 连接状态

**修复方案**：

### 客户端修复 (`public/index.html`)
1. `recallMessage()` 函数添加 socket 连接检查
2. 添加 `permission-denied` 事件监听，显示权限不足提示
3. 更新 `registerMessageRecalledListener()`：
   - 添加 `recall-success` 事件监听，显示成功通知
   - 添加 `recall-failed` 事件监听，显示失败原因
   - 优化"消息已撤回"的显示样式（灰色斜体）

### 服务器端修复 (`server.cjs`)
重写 `message-recall` 事件处理逻辑：
1. 添加所有错误场景的反馈：
   - 用户未登录 → `recall-failed`
   - 无撤回权限 → `permission-denied`
   - 房间不存在 → `recall-failed`
   - 消息不存在 → `recall-failed`
   - 不是自己的消息 → `recall-failed`
2. 撤回成功后发送 `recall-success` 给发送者

**修复文件**：
- `public/index.html`：`recallMessage` 函数和 `registerMessageRecalledListener` 函数
- `server.cjs`：`message-recall` 事件处理

## 修复发送的视频/音频/文件消息自己看不到

**问题描述**：用户发送视频、音频或其他文件类型消息后，自己在聊天界面看不到这些消息。

**根本原因**：
1. 对于图片文件，代码在发送前调用了 `addMessage` 在本地显示
2. 但对于音频和文件（包括视频），代码只通过 socket 发送消息，没有在本地调用 `addMessage`
3. 由于 socket 消息处理会跳过自己发送的消息（`if (data.senderSocketId === socket.id) return`），导致这些消息在本地完全不显示

**修复方案**：

### 客户端修复 (`public/index.html`)
为音频和文件上传添加本地消息显示逻辑，与图片保持一致：
1. 生成消息 ID 和获取用户颜色
2. 上传成功后，先调用 `addMessage` 在本地显示消息
3. 然后通过 socket 发送消息（携带 `clientMessageId`）

**修复位置**：`public/index.html` 文件上传处理逻辑（约第 18664-18720 行）

## 修复通话控制按钮不响应点击事件

**问题描述**：通话控制按钮（静音、关摄像头、结束通话等）点击后没有反应。

**根本原因**：
1. 通话控制按钮的事件监听器在第二个 `onDOMReady()` 回调中注册
2. 第二个 `onDOMReady()` 回调可能因为某些原因没有正确执行
3. 而字幕按钮的监听器在第一个 `onDOMReady()` 回调中，所以能正常工作

**修复方案**：
1. 将所有通话控制按钮的监听器移到第一个 `onDOMReady()` 回调中（与字幕按钮一起）
2. 删除第二个 `onDOMReady()` 回调中的重复代码
3. 添加 `.video-controls` 容器的 `z-index: 10` 确保按钮在最上层

**修复文件**：`public/index.html`
**涉及按钮**：📹 视频开关、🎤 音频静音、📞 结束通话、🔄 切换摄像头、🖥️ 屏幕共享、🔇 降噪、⚙️ 设置

**需求描述**：用户要求改进降噪功能，使其更智能、更有效。

**改进方案**：

### 1. 先进的音频处理算法
- **实时噪声估计系统**：动态检测和建模环境噪声，持续更新噪声配置文件
- **多维度语音概率检测**：基于能量、过零率、峰峭因子、方差等多特征计算语音概率
- **自适应降噪**：根据噪声水平和语音概率自动调整降噪强度
- **频率选择性降噪**：保护重要语音频段（1-4kHz），同时抑制环境噪音
- **软转换处理**：避免硬切换，提供平滑的音质过渡

### 2. 增强的音频处理链
- **3段均衡器系统**：
  - 高频增强滤波器（2500Hz, +3dB）：增强语音清晰度

- **动态范围压缩器**：24dB阈值，10:1比率，平滑音量波动
- **自适应噪声门**：智能检测声音活动，过滤背景噪声
