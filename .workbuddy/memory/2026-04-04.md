## 修复位置权限提示文字误导用户（补充）

**追加修复**：截图确认用户使用 localhost:147（HTTP）访问，位置权限显示"已允许"但仍被浏览器拒绝。  
**根本原因**：Edge/Chrome 在非 localhost 的 HTTP 环境下强制拒绝位置 API，即使权限页面显示"已允许"。  
**修复内容**：
1. `sendCurrentLocation()` 和 `shareRealTimeLocation()` 入口加 HTTP 环境检测，非 localhost 的 HTTP 直接返回并提示改用 HTTPS
2. 错误回调（3 处）细化 `error.code === 1` 的判断，区分 HTTP 强制拒绝和用户主动拒绝两种场景，分别给出精准提示

## 修复位置权限提示文字误导用户

**问题**：用户未拒绝位置权限，但提示显示"您拒绝了位置请求权限"。  
**原因**：`error.code === 1`（PERMISSION_DENIED）不一定是用户主动拒绝，也可能是浏览器/系统/HTTP环境自动拒绝。  
**修复**：将 `public/index.html` 中三处错误提示改为"位置权限被拒绝，请检查浏览器或系统的位置权限设置"，不再指责用户"您拒绝了"。

## 默认关闭病毒检测

将 `server.cjs` 第 67 行 `virusScanEnabled` 的初始值从 `true` 改为 `false`，服务器启动后不再自动启用病毒扫描功能。若病毒扫描模块加载失败，仍会保持禁用状态。
