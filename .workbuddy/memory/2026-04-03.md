### 3. 修复代码块无法撤回的 Bug
**根本原因**：前端 `sendCodeBlock()` 本地预渲染时使用本地临时 ID，发送给服务端后服务端重新生成新 ID 广播。撤回时服务端记录的是真实 ID，但 DOM 里只有临时 ID，导致 `querySelector` 找不到元素。

**修复方案**：
- `server.cjs`：在消息 ID 生成逻辑里，新增支持 `data.clientMessageId`（长度 ≤64），若客户端提供则使用客户端 ID
- `public/index.html`：`sendCodeBlock()` 发送时携带 `clientMessageId: msgId`

**修改文件**：
- `server.cjs`（约第 3528 行附近）
- `public/index.html`（`sendCodeBlock` 函数内 `socket.emit('message', ...)` 处）

### 4. 文档编辑系统安全加固

**修改文件**：`server.cjs`（文档事件处理区域）、`public/index.html`（文档标题输入框）

**加固内容**：

| 安全问题 | 修复措施 |
|---|---|
| 标题无长度限制（XSS 注入风险） | `sanitizeDocTitle()` 函数：去除 HTML 标签、过滤 `< > " ' &`，限制 60 字符 |
| 内容无大小限制（超大数据攻击） | `document-edit` 限制内容最大 200KB |
| 每个房间文档数量无限制（DoS） | `document-create` 限制每个房间最多 30 个文档 |
| 任何人可删除/改名他人文档 | `document-delete`、`document-rename` 只允许文档创建者、管理员、房间创建者操作 |
| 跨房间访问文档 | `document-join`、`document-edit`、`document-save`、`document-delete`、`document-rename` 均校验 `document.roomName === user.roomName` |
| 未加入文档即可编辑 | `document-edit` 校验 `document.users.includes(socket.id)` |
| 创建时接受客户端传入 content | `document-create` 创建时 content 始终为空，不信任客户端 |
| 前端标题输入框无 maxlength | `documentTitleInput` 和 `newDocumentTitle` 均加 `maxlength="60"` |

#### 修复：病毒扫描超时导致文件被误判为病毒

**问题描述**：VirusTotal 扫描超时时，`scanBuffer()` 的 catch 块返回 `{ safe: false }`，导致 server.cjs 将正常文件当作病毒拒绝上传。

**根本原因**：`safe: false` 表示"文件有病毒"，但扫描超时/失败只是"不确定"，不等于"有病毒"。

**修复方案**：将 `scanBuffer()` 和 `scanFile()` 中 catch 块的 `safe: false` 改为 `safe: true`（不确定时默认放行）。

**修复文件**：`utils/virus-scanner.js`（`scanBuffer` 和 `scanFile` 的 catch 块）
